# Архитектура Synthetic Mind Core

## Обзор

Архитектура основана на трёх фундаментальных принципах:

1. **Приватное ядро** — автономная, онтологически замкнутая система.
2. **Проверка** — внешняя и внутренняя согласованность структуры.
3. **Продукт-ориентированность** — каждый агент существует как диагностируемый, используемый продукт.

## Фазово-онтологическая модель

### Фазовый стек (`phase_stack/`)
- **Внимание**: концентрация ↔ рассеяние.
- **Ритм**: внутреннее время как дробный процесс.
- **Переходы**: условия сдвига фазы (ускорение, пауза, затухание).
- **Автоматы**: онтологические конечные автоматы для локального поведения.

### Малоразмерные группы (`ontic_groups/`)
- Атомы опыта, собранные из проекций.
- Когерентность определяет устойчивость смысла.
- Распад группы → иррациональность, «тень».

### Рефлексивная петля (`reflexive_core/`)
- Напряжение между субъективной моделью (`SelfModel`) и входом.
- Незавершённость как механизм удержания иррационального.
- Само-модель обновляется только при достижении фазового равновесия.

## Память и рассуждение

- **Память** — сферическая топология (объём информации).
- **Рассуждение** — торoidal (цикличность, рефлексия).
- Реализовано в `memory/memory_layout.rs`.

## Связь с продуктами

Ядро **не имеет внешних зависимостей**. Продукты подключают:
- `language-module` (MPL) — для лингвистической обработки,
- `meta-ontology-module` (MPL) — для синтеза смысла,
- `runtime` (Apache) — для безопасного взаимодействия с GPL.

> Архитектура гарантирует: **«Если вы убрали продукт — ядро остаётся целым»**.
