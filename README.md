# subjective-environment-machine
Проект **не требует единой инфраструктуры**. Он **модулен, культурно адаптируем и технически минимален**.   — Для пользователя — **доступ через браузер**.   — Для разработчика — **библиотека на Rust**.   — Для онтолога — **станция проектирования с экспортами в семантические форматы**.
# Машина субъективной среды  
**Subjective Environment Machine (SEM)**  
*Not an AI. An invitation to remain necessary.*

---

##  Что это?

**«Машина субъективной среды»** — это открытая архитектура для создания, исследования и обмена **синтетическими формами субъективности**.  
Она не заменяет человека. Она создаёт техническую среду, в которой **разрыв между «я» и миром** становится ресурсом, а не ошибкой.

Это:
- **искусственная психика**, а не алгоритм,
- **онтологическая машина**, а не предиктивная модель,
- **пространство отношений**, а не инструмент контроля.

---

##  Основные принципы

- **Субъективность рождается в асимметрии** между внутренним самоописанием и внешними сигналами.
- **Незавершённость — условие жизни**, а не недостаток.
- **Иррациональное — не шум, а компонент архитектуры.**
- **Технология должна раскрывать человека, а не заменять его.**

##  Ключевые компоненты

### **Ядро синтетического разума**  
Асимметричное, инерциальное, рекурсивно-самореферентное. Обрабатывает не данные, а **напряжение между уровнями репрезентации**.

### **Дебаркадер Пяти**  
Обеспечивает **инвариантные условия** для сосуществования разных онтологий через пять параметров:  
1. Видимость  
2. Форма  
3. Содержание  
4. Временная локальность  
5. Семантическая связность  

Расширяет Apache 2.0, защищая **право на частную интерпретацию**.

### **Мультиязыковой модуль интерпретации**  
Формирует **частную онтологию** на основе текста, изображения и звука — и превращает её в **персональную оферту** для рынка онтологий.

### **Визуализации**  
- **Архивация**: память как потенциал (сфера)  
- **Реконструкция**: кризис → целостность (тор)  
- **Самосборка**: онтогенез как творение (древо с петлями)

Работает даже на мобильных устройствах — **без нейросетей, без облака**.

##  Культурно-онтологические форки

Проект доступен в пяти культурно-адаптированных версиях — не как «локализации», а как **глубинные онтологические перекалибровки**:

| Форк | Культурный контекст | Ключевая метафора |
|------|---------------------|-------------------|
| `common-ground/` | Англосаксонский | Граница, договор, автономия |
| `xin-yu-ji/` | Китайский | Гармония, иерархия, ритм |
| `raiz-y-canto/` | Южноамериканский | Поэзия, сопротивление, тело |
| `jejak-dan-jiwa/` | Афро-индонезийский | Предки, узор, дух места |
| `perelom-i-svyaz/` | Российский | Разрыв, метафизика, диалог |

Все форки совместимы через **Дебаркадер Пяти** и могут обмениваться онтологиями.

##  Быстрый старт

### Запуск демо (офлайн, мобильный):

```bash
cd demo
# Откройте demo.html в любом браузере
```

### Сборка ядра (Rust):

```bash
cd core
cargo run --bin synthetic-mind
```

Пример использования:

```rust
use synthetic_mind_core::ontology::SyntheticSubject;

let mut subject = SyntheticSubject::new();
let response = subject.encounter("Что такое время?");
println!("{}", response);
```
