# Интеграция core144-public в сторонние проекты

Библиотека `core144-public` предназначена для встраивания в приложения, реализующие синтетическое познание на основе онтологических профилей (`onto16`) и проекций `NoemaFast`/`NoemaSlow`.

Это означает:

> ⚠️ **Важно для GPL-проектов**:

## 2. Подключение через Cargo

Добавьте в `Cargo.toml` вашего проекта:

toml
[dependencies]
core144-public = { version = "0.1", path = "../core144-public" }
# или через crates.io/git, если опубликовано


## 3. Базовое использование

rust
use core144_public::{Profile, Core144};

// 1. Загрузка профиля из YAML-байтов
let profile_data: &[u8] = include_bytes!("my_profile.yaml");
let profile = Profile::from_yaml(profile_data)?;

// 2. Инициализация ядра
let mut core = Core144::new(profile)?;

// 3. Обработка стимула (внешнего события или запроса)
let stimulus = "Что делать при неожиданном факте?";
let response = core.process(stimulus)?;

// 4. Генерация онтологической проекции (onto16)
let projection = core.generate_projection()?;


## 4. Совместимость с WASM

Библиотека не использует системные вызовы и совместима с `wasm32-unknown-unknown`.  
Для использования в веб-интерфейсах (например, в ontoCMS) убедитесь, что:

- YAML-профиль загружается как `&[u8]` (например, через `fetch().arrayBuffer()` + Rust-биндинги),
- Все операции происходят в рамках одного потока (WASM-ограничение).

## 5. Поддержка онтологий

- Вход: профиль в формате `onto16` (YAML),
- Внутренняя обработка: `NoemaFast` (внешняя рефлексия) и `NoemaSlow` (внутренняя стабилизация),
- Выход: структурированная проекция, пригодная для фильтрации, логирования или передачи в синтетический транспондер.

## 6. Дальнейшие шаги
